# Project Vision

## Problem Statement
Need a quick way to see the current time in multiple cities at a glance, without opening separate apps or searching.

## Target Users
Personal use — single user (the developer).

## Goals & Key Outcomes
- Display live clocks for multiple cities simultaneously
- Mobile-friendly web app that works well on phone browsers
- Fast load times and minimal overhead
- Deployed and accessible on Vercel

## Constraints
- Must be mobile-friendly (responsive design)
- Deploy to Vercel
- Performance-first — fast initial load, minimal bundle size
- v1 cities are hardcoded: Columbus (OH), New York (NY), Hyderabad (India)

## Timeline
No hard deadline — personal project.

## Success Criteria
- App loads quickly on mobile
- All three city clocks display correct, live-updating times
- Deployed and accessible via Vercel URL

## Out of Scope
- City selection / search UI
- User accounts or persistence
- Multiple themes or customization
